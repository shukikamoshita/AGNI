# システム要件定義書

## システム概要
西川農園のコーポレートサイトは、山梨県甲州市を拠点とした果樹栽培および学生参画型農業の活動を、取引先や外部関係者へ公式に伝えるための「名刺代わり」のサイトです。本サイトは、農園の基本情報、活動背景、関わる人々のストーリーを可視化することで、信頼性の向上とブランド価値の構築を目指します。直接的な売上創出ではなく、情報の集約と信頼構築のインフラとしての役割を担います。

1. システムの目的
   - 解決する課題：農園の公式情報や学生参画の取り組みが体系的に伝えられていない。
   - 目指す状態：URLを共有するだけで、農園の信頼性と活動内容が一目で理解される。
   - 期待される効果：取引先（卸・企業）からの信頼獲得、関係人口の土台作り。

2. 主な機能
   - コア機能：トップページ（第一印象）、農園紹介/ストーリー（背景・想い）、取り組み・プロジェクト紹介（学生参画）。
   - 差別化要素：学生が中心となって運営・参画している「新しい農業の形」の可視化。
   - 最低限の動的機能：お問い合わせフォーム（連絡手段）。

3. 提供する価値
   - 閲覧者（取引先・一般ユーザー）：農園の透明性と信頼性を確認できる。
   - 農園・プロジェクト側：活動を公式に発信でき、ブランド力が向上する。
   - 地域・社会的価値：山梨県甲州市の魅力発信と、学生による地域活性化の事例提示。

4. 実現方法の概要
   - 技術的方針：Next.js + Vercel による静的生成（SSG）中心の構成。
   - 外部連携ポイント：Supabase（問い合わせデータの蓄積・管理）、GitHub（プロジェクト管理）。
   - 将来拡張を阻害しない設計方針：コンポーネントベースの設計により、将来のEC連携やブログ機能追加に対応。

## 情報の前提と仮定
- 使用した入力ファイル：`docs/input/idea.md`
- 不足情報に対して置いた仮定：
    - (仮定) 管理画面は不要とし、データの更新はソースコードの修正および再デプロイで行う。
    - (仮定) お問い合わせフォームの通知機能は、Supabaseの機能または外部サービスを検討する。

## 機能要件
- **トップページ (Must)**: 農園の世界観、キャッチコピー、主要コンテンツへの導線。
- **農園紹介 / ストーリー (Must)**: 西川農園の歴史、栽培への想い、甲州市の環境紹介。
- **取り組み・プロジェクト紹介 (Must)**: 学生参画プロジェクトの具体的内容と活動報告。
- **お問い合わせフォーム (Must)**: 名前、メールアドレス、内容の入力・送信機能。
- **作物紹介 (Should)**: 栽培している桃、葡萄などの紹介（写真・特徴）。
- **メンバー紹介 (Should)**: 農園主や活動に参加する学生の紹介。

## 非機能要件
- **パフォーマンス**: ページ表示3秒以内を目標（静的生成により最適。初期ユーザー少数。）。
- **セキュリティ**: 常時HTTPS化。お問い合わせフォームのスパム対策。
- **運用性**: 学生でもCursorを使用して容易に更新・修正が可能であること。
- **コスト**: Vercel/Supabaseの無料枠を活用し、原則ランニングコスト0円を目指す。

## 技術スタック
- フロントエンド: Next.js (React) - 開発効率と静的生成によるUXを重視。
- 言語: TypeScript - 型安全性の確保によるバグ抑制。
- バックエンド: Next.js App Router - API Routesとしての活用。
- データベース: Supabase (PostgreSQL) - 問い合わせ機能のバックエンドとして。
- 認証: 原則不要 - 公開情報のみの構成。
- ホスティング: Vercel - Next.jsとの親和性と無料枠の活用。

## 開発ロードマップ
- フェーズ0：MVP（トップ、ストーリー、プロジェクト、問い合わせ）の公開。
- フェーズ1：作物紹介、メンバー紹介の追加、スタイリングのブラッシュアップ。

## リスク分析
- **技術的リスク**: Next.js/Supabaseのバージョンアップに伴うメンテナンス（最小限の機能に絞ることで軽減）。
- **運用リスク**: 活動内容の更新頻度が低下する（固定ページ中心の設計にすることで「古さ」を感じさせない工夫）。
- **情報発信上のリスク**: 学生の顔出し等によるプラバシー。

## UI/UXの方向性
- デザインコンセプト：温かみがありつつも洗練された「土の温もり」と「若さ」の融合。
- トーン＆マナー：アースカラー（緑、茶、ベージュ）を基調とし、高品質な写真を多用。
- デバイス方針：モバイルファースト。
